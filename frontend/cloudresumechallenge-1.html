Cloud Resume Challenge: Take 1 & Learning Experience

Cloud Resume Challenge: First Take

Live site: jericho-crc-site.xyz --- password: jkocloudman (yes I know it's as secure as a paper curtain)
GitHub repository: https://github.com/jericholeal/jl-cloud-resume-challenge

Embarking on this project was the first time I ever touched AWS cloud services and resources and hands-on cloud engineering. I dove into it with minimal prior knowledge about building things in the cloud, as the Cloud Resume Challenge originally intended.

What I did and learned in attempt 1:

  • Primarily used the AWS Console and learned how things worked
  • Re-discovered HTML, introduced to CSS and JavaScript
  • Hosted zones, DNSSEC
  • Encryption at rest and in transit
  • Serverless (Lambda, DynamoDB)
  • CDNs, OAC, and CDN caching
  • APIs and HTTP methods
  • JSON
  • How front and backends work together
  • How cloud resources connect, communicate, and fit together as a whole
  • GitHub Actions, CI/CD
  • First time really using Git and GitHub for version control on an actual project
  • Terraform (Infrastructure as Code), as a challenge step was to convert manually-created resources into .tf configurations

Other notes:

  • The resume on the live site of Take 1 is obviously very true, accurate, and realistic. (It was a placeholder that I threw in there while doing all the backend work and I decided to leave it like that when I started my second attempt.)
  • For fun, I decided to do the simple auth page on the frontend to access the main index, plus an access denied page on incorrect password input (I'm aware it doesn't really protect anything, but it was for fun and I wanted to see if I could, so I did)
  • I actually didn't try to deploy the entire thing using Terraform as a later step prescribed. It was at this point that I decided to redo the entire thing (Take 2), but starting with Terraform from the get-go. I'm thinking about revisiting my Take 1 project again to mess around and improve on it with everything that I've learned so far.

What changed between Take 1 and Take 2?

  • Primarily used AWS CLI rather than the console (about 95% of the time) using SSO login
  • Used Terraform from the start rather than at the end
    • I rehashed and improved on a lot of the configurations I made during Take 1
    • I decided to take a modular approach
    • I bootstrapped some key resources (e.g. Route53 hosted zone) and imported them before applying the entire Terraform configuration
  • Redesigned the frontend, adding more pages and repurposing the site as more of a portfolio site rather than a resume
  • Used OIDC roles for GitHub Actions workflows instead of stored credentials as repository secrets

What I learned the second time around:

  • Terraform modular architecture (separate modules, variables & outputs, locals, tfvars, main.tf)
  • Got more comfortable with HCL and writing Terraform configurations
  • Importing existing resources with Terraform
    • I tried terraform apply without importing my existing Route53 zone that I manually provisioned and it caused issues (ACM certificate validation; Route53 creating new hosted zones and trying to validate with the wrong zone and NSs)
  • Resource dependencies
    • Ran into a lot of circular dependency problems (specifically with Route53, CloudFront, and S3)
  • OIDC roles
  • More about least-privilege access
  • That I can do what I set my mind to, even if at the start I'm not sure what I'm doing
