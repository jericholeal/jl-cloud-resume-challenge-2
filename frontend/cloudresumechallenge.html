My Cloud Resume Challenge Attempt(s)

This website is the result of my attempt at the Cloud Resume Challenge originally created by Forrest Brazeal, a well-known cloud architect, writer, speaker, and AWS Community Hero who designed the challenge to help people build real-world cloud skills and give aspiring cloud professionals a structured, hands-on way to showcase their skills with real infrastructure.

This is actually the second version of my Cloud Resume Challenge. Below is an overview, and further down, you can read more about my journey through this.

Cloud Resume Challenge (Take 2)

Live site: jlcrc.cloud
GitHub repository: https://github.com/jericholeal/jl-cloud-resume-challenge-2

Project Overview

  • Deployed a multi-page static website (HTML, CSS, JS) on S3, served via CloudFront with OAC
  • Implemented a session-based visitor counter using API Gateway, Lambda, and DynamoDB
  • Defined fully modular Terraform configurations (ACM, API Gateway, CloudFront, DynamoDB, IAM, Lambda, Route53, S3) with remote state (S3 + DynamoDB lock)
  • Bootstrapped key resources, then imported and managed all infrastructure with Terraform, practicing least-privilege and modular design best practices
  • Built CI/CD pipelines using GitHub Actions + OIDC, with separate workflows for frontend, backend, and Lambda packaging
  • Secured traffic with ACM TLS certificates, custom domain via Route 53, and full DNSSEC setup (KMS-signed, DS record published on NameCheap)
  • Used IAM Identity Center (SSO) for CLI access and AWS Organizations for account and role isolation
  • Planned enhancements: Add CloudWatch alerts, Slack/PagerDuty notifications, IAM policy validation, and CI/CD security gates

Cloud Resume Challenge: Backstory

My long-term goal is to build a career in cloud engineering, so I wanted to do a project where I could just dive into the deep end and get my hands dirty. I found the Cloud Resume Challenge and took the leap with AWS as my cloud provider of choice, this being my first time ever touching AWS and its resources and services. I got my hands on the AWS console and learned about the different resources and how the pieces fit together.

Near the end of this first attempt, I started to realize things that I could've done better and started imagining what I would do if I did it over again. So, I just decided to do it over again, with those things in mind, and here we are now.
